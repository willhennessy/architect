Rundler Pool
Rust · in-memory mempool
Authoritative mempool. Validates, stores, and serves pending user operations. Owns reputation and paymaster tracking state.
Pool API Chain Watcher Validation Pipeline UO Mempools (per EP) Reputation Manager Paymaster Tracker
🔨
Rundler Builder
Rust · bundle assembler
Pulls ops from Pool, proposes and signs bundles, tracks transaction lifecycle, and submits to the chain via public or private relay.
Builder API Assigner Bundle Sender Workers Bundle Proposers Transaction Tracker Transaction Senders Signer Manager

Key Container Relationships
RPC → Pool submit / query user operations RPC → Builder debug / admin control (when enabled) Builder → Pool fetch UOs for bundle construction Pool → Builder push new-UO events to trigger builds Pool → EVM Node chain state, validation simulation Builder → EVM Node bundle simulation, tx submission Builder → Private Relay MEV-protected bundle submission Builder → AWS KMS sign bundle transactions Builder → Redis distributed signer lock (optional)
Deployment modes:  node collocates all three roles in one process (in-process handles).  backend collocates Pool + Builder, keeping gRPC APIs exposed.  rpc / pool / builder run as fully separate processes over gRPC. Entry-point versions v0.6–v0.9 are all handled; each enabled EP gets its own in-memory mempool inside Pool.
